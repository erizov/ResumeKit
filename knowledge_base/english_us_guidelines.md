# English Resume Guidelines - US Market

## Overview

US resumes follow strict conventions focused on brevity, achievements, and ATS compatibility. Understanding these is essential for successful job applications in the United States.

## Key Characteristics

### 1. Personal Information
- **NO Photo**: Photos are not included and may lead to discrimination concerns
- **Minimal Personal Details**: Name, phone, email, LinkedIn, location (city, state)
- **No Personal Info**: Do NOT include: age, date of birth, marital status, photo, gender

### 2. Structure and Order

**Standard Order:**
1. Header (Name, Contact Info)
2. Professional Summary (2-3 lines) OR Objective
3. Work Experience (Most recent first)
4. Education
5. Skills
6. Optional: Certifications, Projects, Languages

**Length:**
- **1 Page**: Standard for most positions (especially <10 years experience)
- **2 Pages**: Acceptable for senior roles (10+ years)
- **3+ Pages**: Generally discouraged unless executive level

### 3. Style and Tone

**Action-Oriented:**
- Start bullet points with strong action verbs
- Focus on achievements, not just responsibilities
- Use metrics and quantifiable results

**Formatting:**
- Clean, professional design
- Standard fonts (Arial, Calibri, Times New Roman)
- Font size: 10-12pt
- Consistent formatting throughout
- White space for readability

### 4. Work Experience

**Format:**
- Company Name | Position | Dates (Month Year - Month Year)
- 3-5 bullet points per role
- Use action verbs: Developed, Implemented, Led, Improved, etc.
- Include metrics: percentages, dollar amounts, time saved

**Example Structure:**
```
Backend Developer | TechCorp Inc. | Jan 2020 - Present

• Developed REST APIs using Python/FastAPI, serving 1M+ requests daily
• Improved application performance by 40% through database optimization
• Led migration to microservices architecture, reducing deployment time by 50%
• Collaborated with cross-functional teams to deliver features on time
```

### 5. Professional Summary

**Best Practices:**
- 2-3 sentences maximum
- Highlight key skills and years of experience
- Tailor to target role
- Include relevant keywords

**Example:**
```
Results-driven Backend Developer with 5+ years of experience building 
scalable web applications. Expert in Python, FastAPI, and PostgreSQL. 
Proven track record of improving system performance by 40%+.
```

### 6. Education

**Format:**
- Degree, Major | University Name | Graduation Year
- GPA only if 3.5+ (optional)
- Relevant coursework only if recent graduate
- Omit high school for college graduates

### 7. Skills Section

**Organization:**
- List technical skills (programming languages, frameworks, tools)
- Group logically or list in order of proficiency
- Include both hard and soft skills
- Match job description keywords

**Format Options:**
- Simple list: Python, FastAPI, PostgreSQL, Docker, Kubernetes
- Categorized: Languages: Python, Java | Frameworks: FastAPI, Spring Boot

### 8. ATS Optimization

**Critical for US Market:**
- Use standard section headers (Work Experience, Education, Skills)
- Include keywords from job description
- Use standard date formats (MM/YYYY or Month YYYY)
- Avoid graphics, tables, or complex formatting
- Use standard fonts
- Save as PDF for submission
- Use .docx for ATS systems that require it

**Keyword Strategy:**
- Mirror language from job description
- Include technical skills mentioned
- Use industry-standard terminology
- Include both acronyms and full terms (API and Application Programming Interface)

### 9. Common Mistakes to Avoid

1. **Too Long**: Keep to 1-2 pages maximum
2. **Including Photo**: Never include photos
3. **Personal Information**: No age, marital status, etc.
4. **Vague Descriptions**: Use specific, quantifiable achievements
5. **Weak Action Verbs**: Avoid "Responsible for", use "Developed", "Led", etc.
6. **Generic Summary**: Tailor summary to each application
7. **Outdated Information**: Remove old, irrelevant experience
8. **Poor Formatting**: Maintain consistent, clean formatting

### 10. Action Verbs for Bullet Points

**Technical:**
- Developed, Designed, Implemented, Built, Created, Architected
- Optimized, Improved, Enhanced, Refactored, Debugged
- Deployed, Configured, Integrated, Migrated, Automated

**Leadership:**
- Led, Managed, Coordinated, Organized, Directed
- Mentored, Trained, Guided, Supervised

**Achievement:**
- Increased, Decreased, Reduced, Improved, Achieved
- Exceeded, Surpassed, Delivered, Completed

### 11. Metrics and Achievements

**Always Include When Possible:**
- Performance improvements (40% faster, 50% reduction)
- Scale (served 1M+ users, processed 100K+ transactions)
- Time savings (reduced deployment time by 50%)
- Cost savings ($X saved, reduced costs by Y%)
- Team size (led team of 5, managed 10+ developers)

### 12. Cultural Notes

- **Results-Focused**: US employers value achievements over tenure
- **Job Hopping**: More acceptable than in other markets (2-3 years is normal)
- **Skills Over Education**: Practical skills often valued more than degrees
- **Networking**: LinkedIn presence is important
- **Cover Letters**: Often required, should complement resume

## Best Practices Summary

1. Keep resume to 1-2 pages
2. NO photo or personal information
3. Use strong action verbs
4. Include quantifiable achievements
5. Tailor to each job application
6. Optimize for ATS systems
7. Use clean, professional formatting
8. Focus on results and impact
9. Match job description keywords
10. Keep summary concise and targeted
